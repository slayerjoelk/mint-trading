=== FILE 1: ceo/croesus.py ===
Croesus — CEO of Mint Trading Company
A single Python script that orchestrates everything.

Imports all 4 agents and infrastructure:
```
from core.db import DatabaseManager
from core.market import MarketInterface
from core.data_pipeline import DataPipeline
from core.news_pipeline import NewsPipeline
from core.risk_manager import RiskManager
from agents.athena.athena_agent import AthenaAgent
from agents.orion.orion_agent import OrionAgent
from agents.sibyl.sibyl_agent import SibylAgent
from agents.janus.janus_agent import JanusAgent
```

Methods:

spawn_agents():
- Initialize all infrastructure (db, market, data, news, risk)
- Create all 4 agents with initial capital:
  - Athena: $25,000
  - Orion: $25,000
  - Sibyl: $15,000 (smaller — news trading is riskier)
  - Janus: $15,000 (vol products are concentrated)
- Print spawn summary with each agent's ID

morning_routine():
- Should be called at market open via cron
- 1. Refresh news pipeline (fetch latest headlines)
- 2. Check market is open
- 3. For each agent: generate_signals()
- 4. For each signal: risk.validate_trade()
- 5. Execute approved trades via market.submit_order()
- 6. Report: which agents generated signals, which were approved, which were executed
- 7. Save report to .hermes/daily-logs/YYYY-MM-DD-morning.txt

evening_routine():
- Should be called at market close via cron
- 1. Get closing prices for all positions
- 2. Close any positions that hit time-based exits (agents can flag these)
- 3. For each agent: learn_from_trades()
- 4. Record daily performance for each agent
- 5. Calculate portfolio-level P&L
- 6. Report: daily P&L per agent, total P&L, MTD, win rates
- 7. Save report to .hermes/daily-logs/YYYY-MM-DD-evening.txt

weekly_review():
- Should be called Sunday via cron
- 1. Agent rankings: sort by Sharpe ratio, win rate, total P&L
- 2. Capital reallocation logic: top performer gets +20% allocation, bottom under 40% win rate gets -50%. Kill agents below 30% win rate with >20 trades.
- 3. Regime analysis: what's the market doing? Which agents benefit?
- 4. Spawn recommendation: if a style is missing, suggest new agent
- 5. Full report saved to .hermes/weekly-reports/YYYY-MM-DD.md
- 6. Print TL;DR summary for Joel's Sunday evening read

run_full_day():
- Runs morning_routine() then waits (returns control to cron)
- The actual market-hours monitoring would be a separate long-running process if needed
