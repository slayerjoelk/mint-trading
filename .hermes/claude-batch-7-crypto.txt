=== FILE 2: agents/mercury/mercury_agent.py ===
Mercury — Crypto Momentum & Structure Agent

Named after the Roman god of commerce and financial gain. Fast, liquid, volatile.

**Style:** Multi-strategy crypto trader combining momentum, mean reversion, and on-chain proxy signals.

**Asset universe:** ['BTC/USDT', 'ETH/USDT', 'SOL/USDT', 'BNB/USDT', 'XRP/USDT', 'ADA/USDT', 'DOGE/USDT', 'AVAX/USDT', 'MATIC/USDT', 'LINK/USDT']

**Strategy (3 sub-strategies):**
1. Trend following: When BTC is above 20/50 SMA and MACD positive → long momentum. Enter on pullbacks to 20SMA, exit on break below 50SMA.
2. Mean reversion: On altcoins (SOL, AVAX, MATIC, LINK), enter when RSI < 30 and price at BB lower band. Exit at BB mid or +5%.
3. BTC dominance rotation: When BTC dominance (proxy via BTC/ETH ratio) rises, rotate to BTC. When it falls, rotate to altcoins.

**Research:**
- Pull 1 year of daily OHLCV for each pair from CryptoMarket
- Compute indicators via data_pipeline (if it works with the data format, otherwise build crypto-compatible versions)
- Test each sub-strategy's historical win rate, Sharpe, and regime dependence
- Save research_report.md

**Signal generation:**
- Check CryptoMarket for latest prices
- Apply all 3 sub-strategies
- Generate signals with confidence scoring
- Limit: 5 signals max per run
- Include specific crypto market context in reasoning

**Learning:**
- After each batch, analyze: which sub-strategy performed best? In what BTC regime (uptrend/downtrend/ranging)?
- Track correlation between crypto pairs — don't enter multiple highly correlated positions simultaneously
- Update thresholds based on evidence
- Contribute insights to SharedKnowledge (import from core.shared_knowledge)

**Journal entries:**
Must include: BTC price context, crypto market cap proxy, BTC dominance estimate, correlation regime note, specific indicator values at entry/exit.

The agent must:
- Inherit from BaseAgent and implement research(), generate_signals(), learn_from_trades()
- Import SharedKnowledge and call add_insight() during learning
- Create its own agent directory under agents/mercury/
- Handle missing crypto data gracefully (empty lists, not crashes)
