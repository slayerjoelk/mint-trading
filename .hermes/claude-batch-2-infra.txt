You are Claude Code. Continue building the Mint Trading Company core infrastructure.

Build these 3 files:

=== File 1: core/news_pipeline.py ===
News ingestion and sentiment analysis. Create a NewsPipeline class that:
- Uses Python's built-in feedparser for RSS feeds (install if needed: pip install feedparser --break-system-packages)
- Feeds: Bloomberg Market News, Reuters Business, CNBC Top News, MarketWatch
- Fallback: if feedparser not available, implement a simple HTTP GET + headline regex extractor
- Methods:
  - fetch_headlines(max_per_source=20) → list of {ticker, headline, source, timestamp, sentiment}
  - extract_ticker_from_headline(headline) → scan headline for known ticker symbols. Use a pre-loaded map of company names → tickers from a CSV or simple dict of top 200 US stocks
  - score_sentiment(headline) → simple keyword-based: positive words (beat, surge, rally, upgrade, buy, growth, profit) = +1, negative (miss, drop, decline, downgrade, sell, loss, crash) = -1. Sum and normalize to -1 to +1
  - get_ticker_sentiment(ticker) → aggregate sentiment for a specific ticker over last 24 hours
  - get_market_sentiment() → overall market sentiment score (average across all headlines)
- Cache headlines in SQLite via db.record_news() to avoid re-processing
- Handle rate limits and connection errors gracefully

=== File 2: core/risk_manager.py ===
Risk controls and position sizing. Create a RiskManager class that:
- Takes a reference to MarketInterface and DatabaseManager
- Config in __init__: default max_position_pct=0.20 (20%), max_portfolio_exposure=0.80 (80%), max_daily_trades=5, max_drawdown_pct=0.25
- Methods:
  - validate_trade(agent_id, symbol, quantity, side, price) → {approved: bool, reason: str, adjusted_quantity: int}
    - Checks: market open? position size within limits? sufficient cash? daily trade count not exceeded? agent not in drawdown?
  - calculate_position_size(agent_capital, price, risk_per_trade=0.02) → int (number of shares). Uses 2% of capital at risk, stop distance from ATR.
  - check_portfolio_exposure(agent_id) → float (current exposure %). Sum of all position values / portfolio value.
  - is_agent_in_drawdown(agent_id) → bool. Check if agent's current drawdown exceeds max_drawdown_pct
  - get_trade_limits(agent_id) → dict of remaining limits for the day
  - kill_switch(agent_id) → liquidate all positions for an agent, set status='killed'
- All calculations use the paper account (not live)

=== File 3: agents/base_agent.py ===
Base agent class that all 4 trading agents inherit from. Create a BaseAgent class with:
- __init__(self, name, style, asset_focus, initial_capital, db, market, data, news, risk):
  - self.agent_id = str(uuid.uuid4())
  - Saves agent to DB via db.insert_agent()
  - Loads or creates config.yaml in agent's own directory
- Properties: name, style, asset_focus, capital, positions (from Alpaca), pnl (calculated), win_rate, total_trades, sharpe_approx
- Core methods (must be implemented by subclasses):
  - research(self) → run research sprint, output a thesis doc and strategy file
  - generate_signals(self) → return list of {symbol, side, quantity, type, reason, confidence}
  - learn_from_trades(self) → review yesterday's trades, update thesis, save new strategy version
- Shared methods (implemented here, usable by all):
  - log_trade(symbol, side, qty, entry, exit, pnl, reasoning) → saves to DB trade + journal
  - get_performance_metrics() → dict with win_rate, total_trades, total_pnl, avg_win, avg_loss, profit_factor, sharpe_approx, max_drawdown, current_drawdown
  - save_strategy(version, description, code_str) → saves strategy .py file + DB record
  - load_strategy(version) → loads strategy from file
  - get_config() / save_config() → YAML config for agent parameters
  - journal_entry(trade_id, analysis, lessons) → writes a detailed post-trade analysis
- The journal entry must be rich — not just "won $50". It must include: what the agent expected, what actually happened, what surprised it, what it will change next time.
